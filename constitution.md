# Constitution: prompt-hub

> 项目宪法。每条都「违反即死」——违反任一条，项目就不再是 prompt-hub。
> 修订必须开 ADR（见 E1）。本文件 6 个月内不应频繁修订。

---

## A. 架构铁律

### A1. 桌面原生 only
本项目必须是桌面原生应用（Tauri / Electron 类），不做 Web 版、浏览器扩展、移动端。
- **证伪**：见到 web build target 或 PWA manifest → 违反
- **依据**：哲学三时间分离机制依赖原生快捷键 + 全屏覆盖窗口，浏览器不可达
- **后果**：失去主形态承载机制，退化为 web 提示词管理器（市场已饱和）

### A2. 本地优先 + 无服务端
所有数据存本地（localStorage / SQLite / 本地文件），不调用自有后端，不依赖云同步。
- **证伪**：见到 fetch 到自有 API / 注册服务端依赖 → 违反
- **依据**：单人工具 + 话术含工作指纹 + spec §8.2 隐私论证
- **后果**：引入运维 / 网络依赖 / 隐私风险——三者任一让产品转型

### A3. 单人单机
不引入用户系统、权限模型、协作机制、分享链接。
- **证伪**：见到 user_id / OAuth / share token → 违反
- **依据**：单人工具的价值最大化依赖高度个性化（spec §8.2）
- **后果**：设计向"通用好用"妥协，失去个性化深度

---

## B. 数据原则

### B1. 资产三层模型不可破
仅 Modifier / Composition / Macro 三层，不引入第 4 层资产类型。
- **证伪**：见到 SuperMacro / SubModifier / 任何第 4 类资产 schema → 违反
- **依据**：三层是观察沉淀的语言系统（spec §3.1）；第 4 层破坏组合可追溯性
- **后果**：数据模型膨胀，UI 无展示位，组合语义混乱

### B2. 协议层与任务层物理分离
AlignmentPhrase 不参与 Modifier 拼接、不入 SOP、不与 Macro 互转、不归属 Scene。
- **证伪**：见到 SOP 步骤含 AlignmentPhrase / Macro 区展示 AlignmentPhrase → 违反
- **依据**：哲学七「先定框、再发任务」的认知节奏依赖物理分离
- **后果**：失去元层协议握手能力，退化为更花哨的提示词管理器

---

## C. 性能底线

### C1. 主形态快捷键唤起 ≤ 200ms 可见
全局快捷键按下到窗口完整可交互必须 ≤ 200ms（P95）。
- **证伪**：benchmark P95 > 200ms → 违反
- **依据**：哲学三「召之即来」——超 200ms 产生迟滞感，主形态心智崩溃
- **后果**：使用者放弃主形态，工具退化为偶尔打开的辅形态

---

## D. 安全/边界底线

### D1. 不内嵌 AI 生成
工具内部不调用 LLM 生成 / 优化 / 分类 / 推荐话术（用户手动粘贴到外部 AI 不算）。
- **证伪**：见到 anthropic / openai / 任何 LLM SDK 用于生成话术 → 违反
- **依据**：手动挡阶段的核心价值是使用者自己思考（spec §8.1 + 哲学一）
- **后果**：使用者消费 AI 通用话术，失去沉淀自己方法论的机会

---

## E. 修订流程

### E1. 宪法变更必须开 ADR
本文件任何一条铁律的修改 / 删除 / 新增，必须先开 ADR 决议；PR 描述需链接 ADR 编号。
- **证伪**：constitution PR 无 ADR 链接 → 违反
- **依据**：宪法变更影响整个项目方向，必须留下决策痕迹
- **后果**：宪法沦为可随意改写的「建议」，失去约束力
