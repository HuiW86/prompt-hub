- **14 tool 全实现**（`crates/prompt-hub-mcp/src/server.rs`）：
  - 5 CRUD：`create_draft` / `get_draft` / `list_drafts`（limit 默认 50、capped 100）/ `update_draft` / `delete_draft`。
  - 3 helper：`bootstrap_from_markdown`（按 `#`/`##` 切段→多 Macro draft）/ `save_conversation_as_macro`（最高频路径，name 缺省取首行）/ `import_json`。

  - 保留 `echo` 作健康检查（第 15 个，不算 14 资产 tool）。
- **`errors.rs`**：`RepoError` → `CallToolResult::error`（is_error=true）携带 what+why+下一个该调工具的纯文本，**不走 JSON-RPC error 通道**（client 会吞、LLM 读不到，§5.4）。
- **`tools.rs`**：`PayloadArg`（4 变体 tagged enum，`schema_version` 默认 1）+ `ProvenanceArg`（`tool_name` 由 server 盖章防伪造，source_app 默认 "Claude Code"）+ `split_markdown_sections`；JsonSchema derive 留在本 crate，不漏 rmcp::schemars 进 repo-core。
- **import_json 加固**（§5.2 部分）：`unchecked_transaction` 全批回滚 / SHA-256 去重（命中 → skip + existing_id，非硬失败）/ batch ≤100 / 单 payload ≤64KB / 每小时 ≤5 次内存配额（进程重启重置）。
- **e2e.rs**：真 spawn 二进制跑 JSON-RPC——CRUD 全生命周期 / 14 tool 全 advertised / 错误是 tool-level 可读 / import 批量去重报告。

## Risks & Decisions

- **read tool filter 偏离 plan §5.3 sketch（有意）**：实际 model 里 `Modifier` 无 phase_id/scene_id、`Macro` 无 phase_id。故 `list_modifiers` 不做 phase/scene filter，`list_macros` 只 filter scene_id。忠于 schema，非 bug。
