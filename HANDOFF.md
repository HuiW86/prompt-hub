## Risks & Decisions

- **migration 串行号**——P2c 必须用 `0008`，db.rs MIGRATIONS 按序登记，勿撞 0007。
- **delete 默认句保护靠 is_default 预查**——`delete_alignment_phrase` 先 SELECT is_default：
  =1 返 `DefaultAlignmentPhraseProtected`，缺失返 `TargetNotFound`，否则 DELETE。前端 store
  乐观删除后若后端拒绝会回滚整个分组快照，默认句在桶内存活。
- **create 强制 is_default=0**——seeded 默认句保持唯一协议默认；partial unique index
  (one is_default=1 per phase) 也会拒绝第二个默认。
- **reorder 完整性靠调用方约定**——`reorder_alignment_phrases(phase_id, ids)` 假设 ids 是该
  phase 完整可见顺序；传部分会让未触及行保旧 order_index。前端永传该 phase 全量，当前安全。
- **B2 物理分离**——AlignmentPhrase 协议层；P3 编辑 UI 禁渗入 Composition/Macro 工作台。
- **编辑 UI 全部 defer 到 P3**——P2a.2 Modifier UI + P2b AlignmentPhrase 编辑面都未做，
  等 P3 自适应布局一起落。Modifier 此前前端零基础，落点槽位由 P3 解决。
- 无 remote——checkpoint 仅本地提交（`git remote` 空）。
