## Risks & Decisions

- **migration 串行号**——0008 已落地（compositions order_index）；下一条 migration 用 0009。
- **Composition update 重写整个 modifier_ids JSON**——`update_composition(id, name,
  modifier_ids)` 全量覆盖 JSON 列，不做增量 diff；前端传完整数组。order_index/phase_id/
  scene_id/usage 不动。
- **reorder 完整性靠调用方约定**——`reorder_compositions(phase_id, ids)` 假设 ids 是该
  phase 完整可见顺序；传部分会让未触及行保旧 order_index。前端永传该 phase 全量，当前安全。
- **per-phase 分区**——order_index 每 phase 重排，与 alignment_phrases 一致；reorder 跨
  phase id 直接拒绝（UPDATE ... AND phase_id=? 命中 0 行返 TargetNotFound）。
- **B2 物理分离**——Composition 是任务层；P3 编辑 UI 禁渗入 AlignmentPhrase 协议层工作台。
- **读侧此前缺失**——Composition 此前只经 promote 落库、从未前端展示。本期补 list 命令 +
  前端类型 + store slice 后才能展示；P3 编辑 UI 依赖此 slice。
- **编辑 UI 全部 defer 到 P3**——P2a.2 Modifier UI + P2b AlignmentPhrase + P2c Composition
  三类编辑面都未做，等 P3 自适应布局一起落。
