## §1 跨形态切换 flow（主 ↔ 辅）

**场景**：用户在副屏开着辅形态做月度 review，主屏正在和 AI 对话，忽然需要快速复制一个 Macro。

```
用户在主屏对话 → ⌥ Space → 主形态弹出（覆盖主屏）
  → 点 Macro 复制 → 主形态自动隐藏（哲学六）
  → 辅形态副屏状态同步更新（usage_count +1）
  → 用户继续 review，看到刚才用的 Macro 高亮在「最近使用区」
```

**关键约束**：
- 主辅形态共享同一数据后端（[[02-constitution#A2]] 本地优先推论——单本地后端，双形态共享）
- 主形态复制触发 UsageRecord 写入 → 辅形态 reactive 监听更新
- 辅形态**不要**因主形态唤起而隐藏（独立窗口，互不影响）

## §2 升级迁移 flow（schema_version 演进）

**场景**：用户使用 v1.x，应用升级到 v2.0（major 升级）。

```
启动 → 检测 schema_version=1.1 vs 代码内置 2.0
  → 显式弹窗：「v2.0 即将到来 / 影响范围 / 会丢失字段 / 是否继续」
  → [取消] → 保持 v1.x 二进制继续运行
  → [继续] → 强制要求导出 v1.x 备份到指定位置
    → 导出成功 → 执行 migrate_1_x_to_2_0
      → 迁移成功 → schema_version=2.0，记录 migration_log
        → 进入 view:home-main
      → 迁移失败 → 回滚到备份 → 错误提示 + 用户介入
    → 导出失败 → 中止迁移，保持 v1.x
```

**关键约束**（[[06-prd#7.7.4]]）：
- minor 升级零感知（无弹窗，无导出要求）
- major 升级强同意（弹窗 + 强制导出 + 显式按钮）
- 任何迁移前自动备份（无备份不迁移）

## §3 数据导入失败 flow

**场景**：用户从其他工具导入 JSON 数据，格式不兼容或含恶意字段（[[06-prd#9.1]]）。

```
view:data-io → 选择 JSON 文件 → 导入按钮
  → schema 校验
    → 通过 → 写入本地数据库 → 提示「成功导入 N 条」
    → 失败 → 显示具体错误（哪个字段 / 哪行 / 为什么拒绝）
      → 提供「查看示例 schema」链接
      → 数据库保持导入前状态（事务回滚）
```

**关键约束**：
- 禁拒键：`__proto__` / `constructor` / `prototype`（防原型污染）
- 单条话术 ≤5000 字符
- 总文件 ≤10 MB
- 必须显示具体错误，不允许"导入失败请重试"模糊提示

## §4 删除资产的 ask-first flow（[[06-prd#8.2-Q1]]）

**场景**：用户在配置面板尝试删除一个使用过 48 次的 Macro。

```
view:config → Macro 列表 → 点删除
  → 二次确认弹窗：
    ┌────────────────────────────────────┐
    │ 「借力最优解」已被使用 48 次          │
    │ 删除后历史 UsageRecord 仍保留（孤儿） │
    │                                    │
    │ [取消]  [标记为 deprecated]  [永久删除] │
    └────────────────────────────────────┘
  → [取消] → 无操作
  → [标记 deprecated] → status=deprecated，UI 隐藏但数据保留
  → [永久删除] → 物理删除 + 内存清零 + 关联 UsageRecord 标记 orphaned
```

**关键约束**：
- 三选项必须始终给（不允许"取消/确认"二选一）
- 高频使用资产（usage_count ≥10）默认推荐 deprecated 而非删除

## §5 快捷键冲突 flow

**场景**：用户配置全局快捷键为 `⌥ Space`，但 macOS Spotlight 也占用此组合。

```
应用启动 → 注册全局快捷键 ⌥ Space
  → 系统返回冲突
  → 弹窗：「快捷键 ⌥ Space 已被其他应用占用」
    → 选项：
      ① 改用备选 ⌥ ⇧ Space
      ② 自定义其他组合（打开配置）
      ③ 暂时只用辅形态（副屏）
  → 选择后写入 config + 重试注册
  → 成功 → 主形态可用
  → 仍失败 → 提示用户手动解除冲突，禁用主形态唤起按钮
```

## §6 首次使用 flow（onboarding）

**场景**：用户首次安装并启动 prompt-hub。

```
启动 → 检测 localStorage 为空
  → 引导页（≤3 屏）：
    第 1 屏：定位说明「这是手动挡阶段的仪表盘」
    第 2 屏：三层资产模型示例 + 哲学七相位带演示
    第 3 屏：是否导入示例数据？
      [是] → 加载内置示例（10 个 Macro / 8 个 Phase / 1 个 SOP）
      [否] → 进入空状态首屏，给出"添加第一个 Macro"引导
  → 注册全局快捷键（见 §5 冲突处理）
  → 进入 view:home-main
```

**反约束**：
- 不强制导入示例（用户的资产应是自己的沉淀）
- 不弹出"教程视频"（哲学一手动挡阶段拒绝过度引导）
- 不要求账号注册（[[02-constitution#A3]]）

## §7 跨设备数据同步 flow（最终一致性）

**场景**：用户在 Mac 主机使用 prompt-hub，希望在 iPad 上只读查看。

```
Mac 上：view:config → view:data-io → 导出 JSON → 保存到 iCloud Drive
  → iPad 应用启动 → 检测 iCloud 同步目录 → 提示「检测到新数据，是否导入」
    [是] → 调用 §3 导入 flow
    [否] → 保持当前数据
```

**关键约束**（[[06-prd#7.4]]）：
- 不做实时同步（单人使用，最终一致性即可）
- iPad 只读为主，不主动写回（避免冲突）
- 通过 Git / iCloud / 手动 USB 同步 JSON，工具不绑定具体同步方式
